Introduction & Document Purpose
This document provides the complete screen-by-screen user flow specification for the Enjoy Series
LMS Admin Application. It covers all three admin login types: Admin, Program Team, and Content
Team. All flows map to the Product Requirements Document (PRD v2) and the Enjoy Series Game
Types Specification.
Key Design Principle
The Content Team should never do double work. Topic Modules are the single authoring surface.
Learning Journeys assemble existing Topic Modules. Games live in a shared repository for reuse. The
Program Team handles operational deployment so the Content Team focuses purely on content creation.
Login Types & Role Definitions
The admin application has three distinct login types. Each login opens a tailored interface showing
only the modules relevant to that role. There are no separate State Admin or City Admin logins —
the Admin manages all organizational levels.

Organizational Hierarchy Management
All organizational management is handled by the Admin login. There are no separate state or city
admin accounts. The Admin manages the full hierarchy: states, cities, schools, and classes. All
entity creation supports both manual entry and CSV/Excel bulk upload.
Universal Upload Pattern
Every entity list screen has “+ New [Entity]” (manual) and “Upload” (bulk). Upload flow: (1) Download
template, (2) Fill data, (3) Upload, (4) Validation preview with green/red rows, (5) Confirm import. Error
rows downloadable for correction.

Content Studio
The Content Studio is the Content Team’s workspace. It contains the Topic Module Builder, Game
Builder, and Game Repository. The Program Team and Admin can view content in read-only mode

Content Architecture
Class → Subject → Topic (per day/session) → Content Blocks (Videos + Games + Quizzes) = 20–25 min
module. Games created in the Game Builder are stored in the Repository and pulled into any Topic
Module for reuse.

Game Builder & Repository
The Game Builder is the Content Team’s dedicated tool for creating adaptive educational games.
The platform supports 14 game types across Maths, English, and Tamil. Games are the primary
vehicle for adaptive learning — the level-based difficulty engine lives inside each game.
Adaptive Logic Location
The adaptive intelligence lives inside the Game Engine. Each game dynamically adjusts difficulty by
serving content from different levels — easier items when the child struggles, harder items when they
succeed. The Topic Module stays simple (linear sequence) while games deliver deep personalization
during active practice.

Game-Type Content Templates & Level Scaling
Below is the content entry template and level-scaling strategy for each game type. This defines
what the Content Team sees when populating a level in Step 2.
01. Drag and Drop / Drag and Fill
Subjects: Maths, English, Tamil
Content Entry Template (per level):
• 	Each item: Draggable Object (image/number/word + audio) + Target Zone label
• 	Maths: object = count image or numeral; target = basket with numeral or place-value label
• 	English/Tamil: object = word card or image; target = blank in sentence or labeled box
• 	Upload images or type text; audio auto-generated or manually uploaded
• 	Set distractor count (extra draggables with no matching target)
Level-Based Difficulty Scaling:
– 	Level 1: 2–3 targets, 3–4 draggables (1 distractor), images only
– 	Level 2: 3–4 targets, 5–6 draggables (2 distractors), images + text
– 	Level 3: 4–5 targets, 6–8 draggables (2–3 distractors), text-heavy, compound words /
multi-digit numbers
02. Tap and Speak
Subjects: English, Tamil
Content Entry Template (per level):
• 	Each item: Image (upload/library) + Expected Word/Phrase + Reference Audio
• 	Toggle: Auto-grading (speech recognition) or Teacher-grading mode
• 	Pronunciation tolerance: Strict / Medium / Lenient
Level-Based Difficulty Scaling:
– 	Level 1: Single common words (3–4 letters), large images, lenient recognition
– 	Level 2: Longer words (5–7 letters), medium tolerance
– 	Level 3: Short phrases/sentences, strict recognition, context images
03. Spot It
Subjects: Maths, English, Tamil
Content Entry Template (per level):
• 	Each item: Item (image/digit/word + audio) + Is Target? flag (yes/no)
